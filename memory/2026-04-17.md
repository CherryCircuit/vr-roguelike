## Slice & Dice Community Modpack Slimming (Graeme's request)

Graeme is a big Slice & Dice fan. He asked to trim Community Modpack V1.42 (332KB, ~3950 lines) so it stops crashing his phone when switching apps.

**File:** `/home/graeme/.openclaw/media/inbound/Community_Modpack_V1.42---e446c406-f828-4199-9a42-723daa1a2424.txt`

**What Graeme wanted cut:**
- NOItems/NOHeroes blocks (vanilla replacements) — keeps original vanilla items
- Items above tier/level 10
- All game modes except Classic
- DEV fights and DEV enemies

**What Graeme wanted kept:**
- All hero color variants (14 colors × 4 levels)
- All custom mod items tier 0-9
- Boss fights
- Classic game mode

**Result:** 332KB → ~295KB (only 11% reduction). The bulk of the file is custom pixel art sprites (113KB, 38%) embedded inline, plus hero variants taking 38% of lines. The phone crash is likely from parsing ~486 inline sprites + 250 cross-references into memory at once.

**Status:** Slimmed version created and sent to Graeme. Waiting for feedback on whether it still crashes. If it does, the next lever is reducing hero color count (14→6 would save ~20%).

**Format notes for future reference:**
- Textmod format: comma-separated `itempool.` sections, `.mn.<name>` groups, `.part.1` chaining, items joined by `+`
- 250 `ritemx.` cross-references (hex IDs) between items
- Hero sections: `.mn.red1`, `.mn.turaco3`, etc.
- Game modes have menus: `.mn.menuDiff`, `.mn.menuHero`, `.mn.menuItem`
- Boss sections: `.mn.Boss4a` through `.mn.Boss20`
