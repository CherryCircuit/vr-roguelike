Most runs show 10-20 MB which is healthy. The 73 MB spikes are intermittent but significant.

## Key Takeaways

1. **The 50ms frame floor is the real target.** Profiled CPU work is only ~2.4ms. The remaining ~46ms is unprofiled or from headless Chrome timing.
2. **GPU render is not the bottleneck** (1.9ms/frame). This is surprising and suggests the real cost is in unprofiled JS work, GC pauses, or Three.js internal updates.
3. **Hellscape is 4x heavier** than other biomes but has improved from the 722-geometry nightmare.
4. **Intermittent memory spikes to 73 MB** need investigation. Could be a leak in combat/restart paths.
5. **Spawn pool exhaustion** occurs in heavy combat when many enemies die and spawn in quick succession.
6. **Game is stable** - zero errors across all runs is excellent.
