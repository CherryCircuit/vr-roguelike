## 1. Can Codey's team create Minecraft Add-Ons?

**Yes, but with an important distinction:**

The Vibecraft GitHub and MaxLeiter blog you linked are about **Java Edition mods** — not **Bedrock Add-Ons** (which is what the Marketplace sells). They're different ecosystems:

| | Java Mods | Bedrock Add-Ons (Marketplace) |
|---|---|---|
| **Language** | Java | JSON + JavaScript |
| **Power** | Full game overhauls | Limited tweaks (mobs, blocks, behaviors) |
| **Can sell?** | ❌ No — free only | ✅ Yes — via Marketplace |
| **Platform** | PC only | Mobile, Console, Windows Store |

**The good news:** AI *can* create Bedrock Add-Ons. CreativeMode.net (one of your links) is literally a tool that generates them from prompts — no coding required. They've already created **260,547+ mods** this way.

**The challenge:** Add-Ons are more limited than Java mods. You can do custom mobs, blocks, items, behaviors, and skins — but not full overhauls like the Create mod.

**Feasibility of "copy + improve":** Moderate. You can't directly port a Java mod to Bedrock, and Add-Ons can't do everything mods can. But for simpler concepts (custom items, mob variants, gameplay tweaks), an AI workflow is absolutely viable.

---

## 2. What language are Add-Ons developed in?

- **JSON** — Defines everything (blocks, entities, loot tables, textures, models)
- **JavaScript** — Optional scripting for advanced behaviors (mini-games, custom events)

Structure:
- **Behavior Pack** (JSON + JS) — Game logic
- **Resource Pack** (JSON) — Visuals, sounds, models
- Packaged as `.mcpack` or `.mcaddon` files

---

## 3. What does it cost to submit?

**$0 — No submission fees.**

You apply to the **Minecraft Partner Program** (free), show a portfolio of community content, and if accepted, you can publish at no cost.

---

## 4. What percentage do creators get?

**~70% to creators** (after platform fees like App Store's 30% cut), with Microsoft/Mojang taking ~30%.

Creators have collectively earned **$500M+** since Marketplace launched. Q1 2025 hit a record $146M in revenue.

**The Play:** Create focused mob packs (e.g., "Ocean Predators", "Nether Beasts") with:

### **2. Furniture Packs That Don't Crash**

**Why:** "Furniture Modern" has **58% 1-star reviews** due to crashes. Top furniture packs (Furniture Life, Furniture Builder) are popular but have performance issues and Bedrock-specific bugs (beds appearing black).

**The Play:** Smaller, themed furniture packs (e.g., "Modern Kitchen", "Cozy Bedroom") that are crash-tested and focus on quality over quantity.

### **3. Utility Add-Ons (Simpler, Lower Risk)**

**Why:** These are easier to build and have dedicated audiences. Popular examples:

- **Hiker's Friend** — Climbing tools, lanterns

**Common complaint:** Backpacks can't auto-fill or open with items in hand. Gravestone clones often have bugs.

**The Play:** Pick one utility concept, execute it flawlessly. Simpler scope = easier for AI to build correctly.

### **4. Starter/Early-Game Add-Ons**

**Underserved niche:** Most packs focus on endgame content. New players or casual players might want:
- Early-game tool upgrades
- Simple quality-of-life tweaks
